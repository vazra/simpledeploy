# Phase 4: Auth & RBAC - Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Multi-user auth with roles (super_admin/admin/viewer), per-app access scoping, JWT sessions for UI, API keys for CLI/LLM tools, bcrypt passwords, login rate limiting, first-run setup.

**Architecture:** Auth package handles password hashing, JWT creation/validation, API key generation/validation. Auth middleware on the API server extracts user from cookie or bearer token, enforces role+app access. Store gets users/api_keys/user_app_access tables. Health endpoint stays public, everything else requires auth.

**Tech Stack:** bcrypt (golang.org/x/crypto), JWT (golang-jwt/jwt/v5), crypto/rand for API key generation, existing store/api packages

#### password.go
```go
package auth

import "golang.org/x/crypto/bcrypt"

const bcryptCost = 12

func HashPassword(password string) (string, error) {
    hash, err := bcrypt.GenerateFromPassword([]byte(password), bcryptCost)
    return string(hash), err
}

func CheckPassword(hash, password string) bool {
    return bcrypt.CompareHashAndPassword([]byte(hash), []byte(password)) == nil
}
```

#### jwt.go
```go
package auth

// JWTManager handles creation and validation of JWT tokens.
// Secret is the signing key. Expiry is token lifetime (e.g., 24h).

type JWTManager struct {
    secret []byte
    expiry time.Duration
}

type Claims struct {
    UserID   int64  `json:"user_id"`
    Username string `json:"username"`
    Role     string `json:"role"`
    jwt.RegisteredClaims
}

func NewJWTManager(secret string, expiry time.Duration) *JWTManager
func (m *JWTManager) Generate(userID int64, username, role string) (string, error)
func (m *JWTManager) Validate(tokenStr string) (*Claims, error)
```

#### apikey.go
```go
package auth

// GenerateAPIKey returns a random API key string prefixed with "sd_".
// Returns (plaintext key, sha256 hash of key).
// Store only the hash. User sees plaintext once at creation.

func GenerateAPIKey() (plaintext string, hash string, error)
func HashAPIKey(key string) string  // sha256 hash for lookup
```

#### ratelimit.go
```go
package auth

// RateLimiter is an in-memory token bucket rate limiter keyed by string (IP).
// Each key gets `limit` tokens per `window`. Tokens refill over time.
// Call Allow(key) before each login attempt. Returns false if rate limited.
// Expired entries are cleaned up periodically.

type RateLimiter struct { ... }
func NewRateLimiter(limit int, window time.Duration) *RateLimiter
func (rl *RateLimiter) Allow(key string) bool
```

#### Migration 003_auth.sql:
```sql
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    role TEXT NOT NULL DEFAULT 'viewer' CHECK(role IN ('super_admin', 'admin', 'viewer')),
    created_at DATETIME NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS api_keys (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    key_hash TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    created_at DATETIME NOT NULL DEFAULT (datetime('now')),
    expires_at DATETIME
);

CREATE TABLE IF NOT EXISTS user_app_access (
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    app_id INTEGER NOT NULL REFERENCES apps(id) ON DELETE CASCADE,
    PRIMARY KEY (user_id, app_id)
);

CREATE INDEX IF NOT EXISTS idx_api_keys_hash ON api_keys(key_hash);
CREATE INDEX IF NOT EXISTS idx_user_app_access_user ON user_app_access(user_id);
```

### Task 3: Auth Middleware

**Files:**
- Create: `internal/api/middleware.go`
- Create: `internal/api/middleware_test.go`

#### Middleware design:

```go
// AuthMiddleware extracts user from JWT cookie or API key bearer token.
// Sets user info in request context. Returns 401 if no valid auth.
// Health endpoint is excluded (always public).

func (s *Server) authMiddleware(next http.Handler) http.Handler

// contextKey type for storing user in context
type contextKey string
const userContextKey contextKey = "user"

type AuthUser struct {
    ID       int64
    Username string
    Role     string
}

func GetAuthUser(r *http.Request) *AuthUser  // extract from context
```

Auth flow:
1. Check `Authorization: Bearer sd_...` header -> hash key, look up in store, get user
2. Else check JWT cookie named `session` -> validate JWT, get user
3. If neither: 401 Unauthorized
4. Store AuthUser in request context

#### App access middleware:
```go
// AppAccessMiddleware checks if the authenticated user can access the app
// identified by {slug} in the URL path. super_admin bypasses.
// Returns 404 (not 403) for unauthorized apps.

func (s *Server) appAccessMiddleware(next http.Handler) http.Handler
```

### Task 4: Auth API Endpoints

**Files:**
- Create: `internal/api/auth.go`
- Create: `internal/api/auth_test.go`

#### Endpoints:

```
POST /api/auth/login        - authenticate, return JWT in httpOnly cookie
POST /api/auth/logout       - clear session cookie
POST /api/setup             - first-run: create initial super_admin (only if no users exist)
```

Login:
- Rate limited by IP (use auth.RateLimiter)
- Request body: `{"username": "...", "password": "..."}`
- Verify password with bcrypt
- Generate JWT, set as httpOnly cookie (SameSite=Strict, Secure if TLS)
- Response: `{"username": "...", "role": "..."}`

Setup:
- Only works if `store.UserCount() == 0`
- Request body: `{"username": "...", "password": "..."}`
- Creates super_admin user
- Returns 409 if users already exist
