The first time SimpleDeploy starts with no users in the database, it shows a setup wizard. You have two paths.

<Tabs>

<TabItem label="Via the web (recommended)">

<Steps>

1. Make sure the service is running:

   ```bash
   sudo systemctl status simpledeploy
   ```

2. Open the management URL in a browser:

   ```
   https://manage.example.com/
   ```

3. The setup screen prompts for a username, password, and optional display name. Submit.

4. The first account is automatically `super_admin`. You land on the dashboard, logged in.

</Steps>

</TabItem>

<TabItem label="Via the CLI">

On the server:

```bash
sudo simpledeploy users create \
  --username admin \
  --password 'CHANGE_ME_LONG_PASSWORD' \
  --role super_admin
```

Then log in at `https://manage.example.com/`.

</TabItem>

</Tabs>

## Strong passwords, please

- 16+ characters, mix of words and symbols.
- Use a password manager. Do not memorize.
- Rotate the password if you ever paste it into a chat or screen-share.

## Issue an API key

API keys are how the CLI and CI systems authenticate. They look like `sd_<random>`.

```bash
sudo simpledeploy apikey create --name "laptop" --user-id 1
# prints the key once. Save it.
```

Or in the dashboard: **Profile &rarr; API Keys &rarr; Create**.

<Aside type="caution">
The full key is shown only once. If you lose it, revoke it and create a new one.
</Aside>

## Roles at a glance

| Role | Can |
|---|---|
| `super_admin` | Everything: users, system, all apps |
| `admin` | All apps + most settings (no user management) |
| `user` | Only the apps explicitly granted to them |

Per-app access for `user` accounts is set under **Users &rarr; (user) &rarr; App access**. See [Roles and permissions](/guides/users-roles/).
