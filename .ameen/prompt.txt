I want to build a really simple and lightweight deployment app similar to dokploy. The goal to make even possible to run in tiny instances with very little overhead. 


- In addition to configuring through the WebUI, I should be able to do everything using code config (instead of setting things up only though UI in dokploy), so that I can use LLM based tools like Claude Code to manage this. 

- run docker compose apps. 
- view system resource usage. 
- view resource usage in app level. 
- view/stream logs from apps/containers. 
- Get resource warnings. 
- backup mechanism for DB's (postgres, backup to s3/blob storage). schedule. recover etc. 
- TLS termination for all apps needed. 
- keep the management really simple. 
- also want to know the rate of API calls and success rate. 
- One binary,minimum memory footprint and size,  minimal dependencies, sqlite db if needed. 
- single node support to get started.
